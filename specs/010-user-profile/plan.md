# Implementation Plan: User Profile Page

**Branch**: `010-user-profile` | **Date**: 2026-07-15 | **Spec**: specs/010-user-profile/spec.md

**Input**: Feature specification from `specs/010-user-profile/spec.md`

## Summary

Add a fully-featured user profile page at `/profile` with view, edit, and password change capabilities. The page is gated behind authentication (redirects to `/login` for unauthenticated visitors). Uses react-hook-form + Zod for edit forms, the existing `FileUpload` component with `variant="circle"` for avatar changes via modal, and the established `api.put`/`api.upload` pattern for persistence. A skeleton loading state is shown while the profile data loads.

## Technical Context

**Language/Version**: TypeScript 6, React 19

**Primary Dependencies**: react-hook-form v7 + @hookform/resolvers + Zod v4, @tanstack/react-query v5, sonner (toasts), existing `api` from `@/lib/api.ts`, existing `FileUpload` from `@/components/ui/file-upload.tsx`

**Storage**: Backend API via `GET /api/user/profile` and `PUT /api/user/profile` (JSON for name/email, multipart for avatar), `PUT /api/user/password` (JSON)

**Testing**: Vitest v4 + @testing-library/react v16 + MSW v2

**Target Platform**: Modern browsers (ES2023)

**Project Type**: Web application (Single Page Application - React + Vite)

**Performance Goals**: N/A ‚Äî single-user view, no throughput targets

**Constraints**: Must follow established form patterns (useEventForm, useMemberForm as reference). Avatar upload must reuse `uploadConfig` from `@/config/index.ts` (5MB max, PNG/JPEG/WebP only). Cookie-based auth via AuthContext.

**Scale/Scope**: Single user profile, max concurrent users equal to site traffic
